# 回答3層モデル

業種を問わず、回答処理を次の3層に分ける。

---

## Layer 1 — 確定情報への回答

ナレッジだけで確実に答えられるもの。AI に自由に考えさせず、登録情報を忠実に案内する。

例:

- 営業時間
- 料金
- 送料
- キャンセル期限
- 施術時間
- 支払い方法
- 予約方法
- 商品の仕様（カタログ記載分）

実装上の扱い:

- 固定回答（fixture）または短いテンプレート
- retrieve ヒットが明確なら AI でもよいが、数値の創作を禁止

---

## Layer 2 — 複数情報を組み合わせた提案

AI の価値が最も伝わる層。複数ナレッジを横断し、条件付きで整理する。

例:

- この条件ならどの商品が合うか
- 4人ならどのコースがよいか
- 初めてならどのメニューが候補か
- 返品可能か、送料負担はどちらか（返品×保証の横断）

実装上の扱い:

- AI ＋ ナレッジ（retrieve → structured output）
- 提案は「候補」として提示し、根拠を明示

---

## Layer 3 — 外部システムまたは担当者への接続

ナレッジだけでは答えられないもの。曖昧に答えず、アクションへ接続する。

例:

- 現在の在庫 / 本日の空席 / 配送状況
- 予約変更 / 返金判断
- 個別の健康状態 / クレーム

接続先の例:

- 在庫 API を確認（デモはモック）
- 予約画面を開く
- 追跡番号を入力
- スタッフへ引き継ぐ
- 電話 / 問い合わせフォーム

実装上の扱い:

- テンプレートフロー＋モックデータ、または有人シート
- 「あります」「空いています」の断定をナレッジだけでしない

---

## 回答方式の分担表

| 回答方式 | 適した質問 |
|----------|------------|
| 固定回答 | 営業時間、支払い方法、アクセス、予約方法 |
| テンプレート回答 | 配送確認、予約変更、注文キャンセル |
| AI＋ナレッジ | 商品比較、コース提案、条件横断 |
| AI＋業務API（モック可） | 在庫、空席、配送状況、注文状態 |
| 有人対応 | クレーム、例外、健康相談、返金判断 |

### 配送状況の流れ（Layer 3 例）

```text
配送状況を確認
→ 注文番号を要求
→ 配送API（モック）を呼ぶ
→ 現在地と予定日をテンプレート表示
```

### 横断回答の例（Layer 2）

```text
返品期限を過ぎたが初期不良のようです。保証と返品のどちらで対応できますか？
→ 返品規定 ＋ 保証規定を AI が横断して整理
```
