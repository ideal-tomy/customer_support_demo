# 在庫・入荷ルール

| 項目 | 値 |
|------|-----|
| documentId | `inventory-rules` |
| タイトル | 在庫・入荷ルール |
| 主な Layer | 1（表示の意味）/ 3（実在庫は API） |

---

## この文書の役割

サイト上の「在庫あり／残りわずか／取り寄せ」の**意味**を説明し、AI がリアルタイム在庫を断定しないためのガード文書。

### 確定値（`data/sample/retail-commerce-v1/documents/inventory-rules.json`）

- 「在庫あり」「残りわずか」「お取り寄せ」「予約販売」は表示時点の目安
- カート投入 ≠ 在庫確保。チャットで「在庫があります」と断定しない
- お取り寄せ目安: 3〜7営業日（商品により異なる）
- 予約販売は入荷後発送
- **明後日までのお届け可否は在庫＋配送日数の両方確認が必要**（着日断定禁止）

---

## 入れるとよい内容

- 在庫表示の種類と意味（あり／わずか／お取り寄せ／予約販売）  
- 取り寄せの目安日数（例: 3〜7営業日）※断定しすぎない幅  
- 予約商品の扱い（入荷後発送）  
- 「カートに入った＝確保」ではないこと  
- 実在庫確認は「在庫を確認」アクション／システム連携が必要であること  
- デモ用: 在庫 API 未接続時の案内文言  

---

## 推奨セクション構成

1. 在庫表示の見方  
2. 取り寄せ・予約商品  
3. 在庫確認が必要なケース  
4. よくある誤解  

---

## この文書が効く質問

- 明後日までに届けられる商品はあるか（配送ガイドと併用、在庫断定禁止）  
- 取り寄せとは何か  
- 今すぐ買えるか  

---

## 書いてはいけない点

- 具体 SKU の「現在○個」  
- 「必ず明日届く」等の確約
