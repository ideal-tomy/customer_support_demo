# カスタマーサポートAI — 要件定義

**Demo ID:** `customer-support`  
**Demo Type:** Chat / FAQ Knowledge / Human Handoff Visualization  
**バージョン:** v0.1  
**関連:** `customer_support_demo_definition.md`, `extraction_from_internal_knowledge.md`, `ai_customer_center_chat_mockup.html`

---

## 1. 背景

企業のカスタマーサポートでは、返品・配送・請求などの定型問い合わせが多く、FAQ やポリシーに答えがある一方で、個別事情は有人対応が必要になる。

本デモは、顧客向けチャットで「ナレッジに基づく自己解決」と「担当者への橋渡し」を体験してもらう。

---

## 2. コンセプト

> FAQ・ポリシー・カタログを根拠に、顧客チャットで結論と次の一手を素早く示し、必要なら有人へ渡せる体験を提供する。

体験者はサンプルFAQで価値を理解したあと、自社の FAQ / カタログテキストを投入して挙動確認できる。

---

## 3. 対象ユーザー

- Primary: CS責任者・EC/サポート企画・AI導入検討者（商談体験）
- Secondary: 営業・パートナー（デモ同行）

---

## 4. Must

- 顧客向けチャットUI（キャラクターヘッダー、クイックリプライ、構造化回答）
- サンプルFAQパック（配送 / 返品・交換 / 請求）
- Core 経由の AI（Sample / BYOK / Managed Trial）
- 選択中ナレッジのみを根拠にする
- マイFAQ / カタログ投稿（.md / .txt / 貼り付け、セッション内）
- 有人相談のデモ演出
- PC / モバイル対応
- 体験コード取得は共通ポータルへ誘導

## 5. Should

- 参考案内へのジャンプ
- 回答ソース表示（fixture / LLM）
- 複数業界向けサンプルパック追加

## 6. Won't（初期）

- 実注文・会員データの照会
- 本番有人キュー / CRM 連携
- 本番機密の永続保存
- SharePoint 等との常時同期
- 社内規程ビューアの全面再現

---

## 7. 成功条件

1. 返品・配送・請求の代表質問に根拠付きで答えられる  
2. 担当者相談導線が分かる  
3. 自社FAQを入れて同じUIで試せる  
4. AIがナレッジ外の約束をしない  
5. 社内ナレッジデモとは別URL・別ブランドで体験できる
